Some thoughts about Java development for Android.

Fragment Arguments

A number of operations or data flows are of interest:
  • Attach data when starting a new Fragment/Activity
  • Retrieving extra/data from Activity (lazy way)
  • Retrieving extra/data by attaching it to Fragment
  • Obtain results back from a Fragment

First of all, to start a Fragment/Activity from a given Fragment, for example OnClick, one uses the new Intent(). Nothing further is needed as the new Activity itself includes the code to start the Fragment. Code to provide the possibility of attaching an Extras/data resides in the Activity file: A static method to construct a new intent together with an putExtra. The static method is then called for actually starting the new Activity/Fragment. Here the Extra data is loaded.

To retrieve the Extra/data, in it's simplest form, it can be obtained directly from the activity: (typeCast) getActivity().getIntent().getSerializableExtra(ID)

. This is however from a modularity point of view not a very elegant solution.

A nicer way is to attach the arguments to a Fragment. To do this, a static method newInstance() is added to the Fragment class to which the data shall be attached. This method takes as an argument the data that shall be included into the Fragment Bundle. Here the createFragment method in the calling Activity has to be adapted to use the new static method. Finally, the arguments are obtained in the Fragment by using the getArguments().getSerializable(ID) method.
